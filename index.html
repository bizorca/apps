Why I'm building all of this

I've spent years doing tax and accounting work — advising small business owners, watching them struggle with cobbled-together spreadsheets and off-the-shelf software that almost fits their workflow but never quite does. The gap between "there's probably an app for this" and "there's actually a good app for exactly this" is enormous. I decided to start closing that gap one project at a time.

I'm doing this with Claude Code, Anthropic's AI coding tool. I'm not a classically trained software engineer. I'm a tax accountant who learned to code — which, it turns out, is a pretty useful combination when you're building business tools. I understand the problem domain. I understand the user. And now I can ship working software in days instead of months.

Every app on this page is something I built because a real problem needed solving. Some are SaaS platforms. Some are games. Some are tools I use myself. All of them were built lean, without a team, without a seed round, and without a year-long roadmap. Just a clear problem, a working solution, and a deploy script.
